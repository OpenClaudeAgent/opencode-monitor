# Maintenance Directory

Ce dossier contient les documents de maintenance, rapports d'audit et analyses techniques qui ne font pas partie du code de production.

## Structure

### `reports/`
Rapports de code review et analyses de Sprint:
- **SPRINT0-*** - Rapports de review du Sprint 0 (quantitatif, architecture, tests)
- **DQ-***-COMPLETION.md - Summaries de completion des user stories
- **BENCHMARK-*.md - Rapports de performance
- **ARCHITECTURE-REVIEW-*.md - Analyses d'architecture

### `audit-reports/`
Audits de données et analyses de complétude:
- Audits de données (coverage, completeness)
- Rapports d'architecture système
- Analyses de qualité des données

## Usage

Ces documents servent à:
- Code reviews et quality gates
- Documentation des décisions techniques
- Traçabilité des audits
- Référence pour les mainteneurs

**Note**: Ces fichiers ne sont pas nécessaires au runtime de l'application.
